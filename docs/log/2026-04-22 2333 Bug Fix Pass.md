## Full notes

### Music cutout after phase 4 (root cause)
`MusicSystem.startMusic` only calls `.play()` on layered sounds the first time `layeredStarted` flips to `true`. Tracks that loaded later via `warmMusicCache` (drums-3, bass-3, gameSynth) were registered into `layeredSounds` by `ensureLayeredSounds` but never started, so when phase 4's mix transition tweened their volumes from 0 → target the underlying sounds were silent because they had never been played.

Fix: inside `ensureLayeredSounds`, when a newly-discovered track is added after `layeredStarted` is true, kick it off immediately at volume 0 with `loop: true`. From there the existing fade-mix logic just works.

```ts
if (layeredStarted && !scene.sound.locked && !sound.isPlaying) {
  sound.play({ delay: MUSIC_START_DELAY_S, loop: true, volume: 0 });
}
```

### Bass deferred to arena + sync reset
- Changed `MISSION_LAYERED_MIX` so `bassOne` is `0` on MissionSelect (was `1`). Menu/MissionSelect now stays on `menuSynth` only.
- Added `resyncGameplayMusic(scene)` to `MusicSystem`: stops and replays each non-menu layered track at volume 0, so they all start fresh from loop position 0. Called from `GameScene.create()` immediately before `setGameplayMusicForPhase(this, 1)`. This means bass-1 is at beat 0 when the entry gate countdown fires, instead of at whatever drift position it had reached running silently from menu boot.

Open question on phase length vs. track length deliberately not addressed — see Known Issues.

### MissionSelect brief overrun
The mission card brief is anchored at `setOrigin(0, 1)` and grows upward from `footerY`. When both the label (top, growing down) and brief (bottom, growing up) wrapped to multiple lines they collided.

Fix in `MissionSelectScene.drawCard`: after both texts are created, compute `briefAvailableHeight = footerY - labelBottom - 2`, use `getWrappedText()` to measure, and if needed slice the wrapped lines and ellipsize the last kept line. Font size unchanged so flavor text stays readable when there's room.

### Hidden debug phase jump
Re-added `debugJumpToPhase(phase)` (lifted from commit 4b368a6). No pause-menu UI. Two access paths, both wired in a new `installDebugPhaseShortcuts()` called from `create()`:
- Keyboard: `Shift+1` … `Shift+9` jumps to that phase, `Shift+0` jumps to phase 10. Reads `event.code` so `Digit1` and `Numpad1` both work; ignores combos with Ctrl/Alt/Meta.
- Console: `window.bitpJumpToPhase(n)` for ad-hoc use; cleared on scene SHUTDOWN.

`scoreRecordingBlocked = true` is set on jump so debug runs can't pollute the leaderboard. The systems-side `extractionSystem.debugSetPhase` and `difficultySystem.debugSetPhase` were already in place.

### Beam SFX
- Added `beamFire` to `SfxSystem` (`SFX_NAMES`, `SFX_KEYS`, `SFX_PATHS`, `SFX_BASE_VOLUME`). Path points at `audio/bomb.mp3` as a placeholder; comment in source flags the swap.
- `BeamHazard` now stores its scene and plays `beamFire` on the warning→lethal transition (`!wasLethal && lethal`) with `rate: 1.6` and `detune: 600` to make the bomb sample feel laser-y.
