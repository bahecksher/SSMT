## Full notes

### Debug pause menu removal
Removed the DEBUG PHASE divider, title, phase chip buttons, and debug status text from `GameScene.showPauseMenu`. The mission section now anchors directly to the FX volume slider instead of the old `phaseRowTwoY`.

The orphaned `createPauseChipButton` and `debugJumpToPhase` methods were also deleted since removing the UI made them unused (TypeScript `noUnusedLocals` caught them). The `scoreRecordingBlocked` field and its checks in the scoring paths are left intact — always `false` now, harmless, removing them would creep into a refactor.

`extractionSystem.debugSetPhase` and `difficultySystem.debugSetPhase` remain in those systems even though nothing now calls them; public methods don't trigger the unused-declaration warning and deleting them was out of scope.

### Offline handling
`LeaderboardService.ts` now exports `isOnline()` (wrapping `navigator.onLine`) and short-circuits all five network functions: `fetchLeaderboard`, `fetchCorporationLeaderboard`, `fetchBiggestLoss` return empty/null, `submitScore` and `submitLoss` become no-ops. Local `BEST` tracking via `SaveSystem.saveBestScore` was already offline-safe and unchanged.

`MenuScene.loadLeaderboard` checks `isOnline()` before firing requests and displays `OFFLINE — GLOBAL BOARDS UNAVAILABLE` in the status text area when offline. The existing `.catch` → "OFFLINE" path still handles transient failures when a request does go out and fails.

### Vite base path
Changed `base` for production builds from hardcoded `/SSMT/` to `./`, gated behind an env var so GitHub Pages builds can still be produced:
```ts
base: command === 'build' ? (process.env.VITE_PAGES_BUILD ? '/SSMT/' : './') : '/',
```
Dev server remains on `/`. Verified the built `dist/index.html` references `fonts/...` and `assets/...` without a leading slash, which Tauri's webview needs.

### Tauri scaffolding
Installed `@tauri-apps/cli@2.10.1` as a devDependency. Scaffolded `src-tauri/` by hand (rather than `tauri init`, which is interactive):
- `tauri.conf.json` — 1280×720 resizable window, product name "Slick's Salvage", identifier `com.brett.slicksalvage`, bundle targets `msi` + `nsis`, CSP set to `null` for now (tighten later once we confirm Supabase + Phaser WebGL work end-to-end).
- `Cargo.toml` — Tauri 2 with `tauri-plugin-opener`, LTO + codegen-units=1 + opt-level=s for a small release binary.
- `build.rs`, `src/main.rs`, `src/lib.rs` — standard Tauri 2 layout with a `mobile_entry_point` stub for future iOS/Android.
- `capabilities/default.json` — `core:default` + `opener:default` (we don't need any extra permissions yet; Supabase calls are plain fetch, not Tauri commands).
- Icons generated by converting `public/company-logo.gif.gif` to a 1024×1024 PNG via PowerShell/System.Drawing, then running `npx tauri icon src-tauri/app-icon.png` which produced full Windows/macOS/Linux/iOS/Android icon sets. Placeholder quality — worth revisiting with a proper square logo before public release.
- `.gitignore` for `target/` and generated schemas.

Added npm scripts: `tauri`, `tauri:dev`, `tauri:build`.

### Verification
`npm run build` passes cleanly. No TypeScript errors after removing the two unused methods.

### Open questions / known risks
- Placeholder icons derived from the company-logo GIF — fine for a smoke test, worth replacing with a crisp 1024×1024 PNG before public itch listing.
- CSP is `null` to avoid breaking Supabase + Phaser WebGL in the packaged webview. Should be tightened to an allowlist after the first successful build confirms everything works.
- `navigator.onLine` is optimistic — reports `true` if any network interface is up. Supabase reachability failures still fall through to the existing `console.warn` handling.
- `src-tauri/target/` will be large (~hundreds of MB) and must stay gitignored.
- No code signing — Windows SmartScreen will show an "Unknown Publisher" warning on first run. Acceptable for itch distribution; revisit with a cert if piracy/trust becomes a friction point.
