# Plan revision - Mission System Economy
_Created: 2026-03-29 0105_
_Revises: docs/plans/2026-03-29 0053 Plan revision - Mission System Economy.md_

## What changed
- Slick's payout progression is removed in favor of a flat `60/40` split.
- MissionSelect economy copy is reduced to a single wallet line above the favor grid.
- Favor prices move into a high-cost range so even basic favors require several successful exfils.

## Why
- The progressive deal copy was adding complexity without helping the player make better pre-run decisions.
- The MissionSelect economy header still felt too dense.
- Favors were still reading as cheap toggles instead of meaningful spend decisions.

## Updated approach
- Simplify the shared payout helpers to a fixed split and remove the deal-progression UI.
- Reduce MissionSelect wallet copy to one clear header line.
- Raise all favor costs sharply, while keeping the existing company-based pricing differences.
