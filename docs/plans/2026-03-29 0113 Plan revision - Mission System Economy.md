# Plan revision - Mission System Economy
_Created: 2026-03-29 0113_
_Revises: docs/plans/2026-03-29 0105 Plan revision - Mission System Economy.md_

## What changed
- Mission rerolls now need a stacking wallet-credit cost.
- The wallet header should be more visually prominent.
- Favor-card title copy should show liaison name without the role suffix.

## Why
- The wallet line was still easy to miss.
- The liaison role was adding clutter without improving decisions.
- Free rerolls made it too cheap to cycle for better contract mixes.

## Updated approach
- Increase the wallet header text size above the favor grid.
- Simplify favor-card title text to `company // liaison`.
- Charge `200c`, `400c`, then `600c` for rerolls within the current MissionSelect visit while still using the existing reroll stock count.
